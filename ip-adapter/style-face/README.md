# IP-Adapter (стиль + лицо) 🎭👤✨

Продвинутый пайплайн для генерации изображений с использованием IP-Adapter. Позволяет комбинировать стиль из одного референсного изображения и лицо из другого, создавая уникальные портреты с контролируемым внешним видом и стилизацией. 🖼️+🎨+👤=✨

## Описание работы пайплайна 🔄

Пайплайн использует IP-Adapter для переноса стиля и лица из референсных изображений:

### Этап 1: Загрузка моделей и референсов 📥
1. **Загрузка базовой модели** 📦 — `CheckpointLoaderSimple` загружает чекпоинт (например, `RealVisXL_V5.0_Lightning_fp16.safetensors`)
2. **Загрузка IP-Adapter** 🎛️ — `IPAdapterUnifiedLoader` интегрирует IP-Adapter в модель (режим `PLUS (high strength)`)
3. **Загрузка изображения стиля** 🎨 — `LoadImage` загружает референс для стиля (общая композиция, цвета, атмосфера)
4. **Загрузка изображения лица** 👤 — второй `LoadImage` загружает референс для лица (портрет с нужными чертами)

### Этап 2: Кодирование референсов 🔢
5. **IPAdapterEncoder (стиль)** 🎨 — преобразует изображение стиля в эмбеддинги (векторные представления)
6. **IPAdapterEncoder (лицо)** 👤 — преобразует изображение лица в эмбеддинги
7. **Настройка веса стиля** ⚙️ — в `IPAdapterEncoder` для стиля установлен `weight = 0.59`
8. **Настройка веса лица** ⚙️ — в `IPAdapterEncoder` для лица установлен `weight = 0.5`

### Этап 3: Комбинирование эмбеддингов 🔗
9. **IPAdapterCombineEmbeds (позитивный стиль + лицо)** ➕ — объединяет позитивные эмбеддинги стиля и лица методом `average` (среднее)
10. **IPAdapterCombineEmbeds (негативный)** ➖ — объединяет негативные эмбеддинги методом `add` (сложение) для усиления подавления нежелательных элементов

### Этап 4: Применение IP-Adapter и генерация ✨
11. **IPAdapterEmbeds** 🎯 — применяет комбинированные эмбеддинги к модели
    - `weight`: сила влияния (`1`)
    - `weight_type`: тип взвешивания (`V only`)
    - `start_at`: начало применения (`0`)
    - `end_at`: конец применения (`1`)
12. **CLIPTextEncode (позитивный)** ✅ — текстовый промпт для контроля содержимого
13. **CLIPTextEncode (негативный)** ❌ — негативный промпт для исключения артефактов
14. **EmptyLatentImage** 🖼️ — создает пустой латент (разрешение `1024x1024`)
15. **KSampler** 🎲 — выполняет генерацию с учетом IP-Adapter и текстовых промптов
16. **VAEDecode** 🔓 — декодирует латент в итоговое изображение
17. **SaveImage** 💾 — сохраняет результат с префиксом `ipadapter_output`

## Быстрый старт 🚀

1. Загрузите пайплайн в ComfyUI (файл `IP-Adapter (стиль + лицо) (4).json`) 📂
2. Убедитесь что у вас установлены необходимые ноды: **ComfyUI_IPAdapter_plus** 📦
3. В `CheckpointLoaderSimple` укажите путь к вашему чекпоинту ⚠️
4. В первом `LoadImage` загрузите изображение для стиля (композиция, цвета) 🎨
5. Во втором `LoadImage` загрузите изображение лица (портрет) 👤
6. При необходимости отредактируйте текстовые промпты ✏️
7. Настройте веса:
   - Вес стиля (`0.59`) — регулирует влияние стилевого референса
   - Вес лица (`0.5`) — регулирует влияние лица
   - Общий вес IP-Adapter (`1`) — сила всего IP-Adapter механизма
8. Нажмите "Queue Prompt" для генерации ▶️
9. Результат появится в папке `ComfyUI/output` с префиксом `ipadapter_output` ✨

## Важные замечания ⚠️

- Для работы пайплайна необходима кастомная нода: **ComfyUI_IPAdapter_plus** 📦
- IP-Adapter модели необходимо скачать отдельно и поместить в папку `ComfyUI/models/ipadapter/` 📁
- Для разных базовых моделей (SD1.5/SDXL) требуются соответствующие IP-Adapter модели 🔄
- Баланс весов стиля и лица позволяет контролировать, что будет доминировать в итоговом изображении ⚖️
- Метод комбинирования `average` для позитивных эмбеддингов дает плавное смешение, `add` для негативных — усиленное подавление 🎯
- Для лучшего результата рекомендуется использовать референс лица с четкими чертами и хорошим освещением 📸
