# IP-Adapter (стиль + лицо) 🎭✨

Пайплайн для переноса макияжа и стиля с одного изображения на другое с сохранением лица и позы целевой девушки. Использует IP-Adapter для точного контроля над переносом стиля. 🖼️+🎨=✨

## Описание работы пайплайна 🔄

Пайплайн комбинирует стиль из одного референсного изображения и лицо из другого, создавая портрет с сохранением оригинальной позы:

### Этап 1: Загрузка моделей и референсов 📥
1. **Загрузка базовой модели** 📦 — `CheckpointLoaderSimple` загружает чекпоинт (например, `RealVisXL_V5.0_Lightning_fp16.safetensors`)
2. **Загрузка IP-Adapter** 🎛️ — `IPAdapterUnifiedLoader` интегрирует IP-Adapter в модель (режим `PLUS (high strength)`)
3. **Загрузка изображения стиля** 🎨 — `LoadImage` (нода 6) загружает референс для стиля и макияжа (например, брюнетка)
4. **Загрузка изображения лица** 👤 — `LoadImage` (нода 5) загружает референс для лица (целевая девушка, поза которой сохраняется)

### Этап 2: Кодирование референсов 🔢
5. **IPAdapterEncoder (стиль)** 🎨 — преобразует изображение стиля в эмбеддинги с весом `1.2` (сильное влияние)
6. **IPAdapterEncoder (лицо)** 👤 — преобразует изображение лица в эмбеддинги с весом `0.8` (сохранение узнаваемости)
7. **VAEEncode** 🔄 — кодирует изображение лица в латентное пространство для сохранения позы

### Этап 3: Комбинирование эмбеддингов 🔗
8. **IPAdapterCombineEmbeds** ➕ — объединяет эмбеддинги стиля и лица методом `average` (сбалансированное смешение)

### Этап 4: Применение IP-Adapter и генерация ✨
9. **IPAdapterEmbeds** 🎯 — применяет комбинированные эмбеддинги к модели:
   - `weight: 1.3` — общая сила влияния
   - `weight_type: "strong style transfer"` — режим для мощного переноса стиля
   - `K+V` — влияние на ключи и значения для лучшей цветопередачи
   - `start_at: 0`, `end_at: 1` — работа на всех шагах
10. **CLIPTextEncode (позитивный)** ✅ — текстовый промпт для контроля композиции
11. **CLIPTextEncode (негативный)** ❌ — негативный промпт для исключения артефактов
12. **KSampler** 🎲 — выполняет генерацию с `denoise: 0.45` (чистота без галлюцинаций)
13. **VAEDecode** 🔓 — декодирует латент в итоговое изображение
14. **SaveImage** 💾 — сохраняет результат с префиксом `ipadapter_output`

## Используемые ноды (переменные для настройки) 🎛️

### Параметры генерации ⚙️

#### `VAEEncode` (нода 17) 🔄
Кодирует изображение лица в латент для сохранения позы. Не требует настройки.

#### `KSampler` (нода 12) 🎲
Основной сэмплер.

| Поле | Описание | Пример значения |
|------|----------|-----------------|
| `seed` | Сид генерации | `356988326714625` или `randomize` |

## Быстрый старт 🚀

1. Загрузите пайплайн в ComfyUI (файл `IP-Adapter (стиль + лицо) (11).json`) 📂
2. Убедитесь что у вас установлены необходимые ноды: **ComfyUI_IPAdapter_plus** 📦
3. В `CheckpointLoaderSimple` укажите путь к вашему чекпоинту ⚠️
4. В `LoadImage` (нода 6) загрузите изображение для **стиля и макияжа** 🎨
5. В `LoadImage` (нода 5) загрузите изображение **целевой девушки** (её поза сохранится) 👤
6. При необходимости отредактируйте текстовые промпты ✏️
7. Настройте ключевые веса:
   - **Нода 8 (стиль)**: `1.2` — сила переноса макияжа
   - **Нода 9 (лицо)**: `0.8` — сохранение узнаваемости
   - **Нода 11**: `weight: 1.3` + `"strong style transfer"` + `K+V`
8. Нажмите "Queue Prompt" для генерации ▶️
9. Результат появится в папке `ComfyUI/output` с префиксом `ipadapter_output` ✨

## Важные замечания ⚠️

- Для работы пайплайна необходима кастомная нода: **ComfyUI_IPAdapter_plus** 📦
- IP-Adapter модели необходимо скачать отдельно и поместить в папку `ComfyUI/models/ipadapter/` 📁
- Для разных базовых моделей (SD1.5/SDXL) требуются соответствующие IP-Adapter модели 🔄
- Пайплайн сохраняет **позу и лицо** целевой девушки, перенося **макияж и общий стиль** 🎯
- Для полного перекрашивания волос рекомендуется использовать **второй проход с инпейнтингом** (Hires Fix + маска на волосы) 💇‍♀️
- Значение `denoise: 0.45` обеспечивает чистоту без галлюцинаций — не рекомендуется поднимать выше `0.6` ⚡
