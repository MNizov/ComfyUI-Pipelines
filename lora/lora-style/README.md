# LoRA  🎨⚙️✨

Пайплайн для генерации изображений с использованием LoRA (Low-Rank Adaptation). Позволяет применять обученные LoRA-модели для изменения стиля, персонажей или элементов изображения без необходимости дообучения основной модели. 🖼️➡️🎭

## Описание работы пайплайна 🔄

Пайплайн реализует стандартный процесс генерации с применением LoRA:

1. **Загрузка базовой модели** 📦 — `CheckpointLoaderSimple` загружает основной чекпоинт (например, `RealVisXL_V5.0_Lightning_fp16.safetensors`)
2. **Загрузка и применение LoRA** 🎭 — `LoraLoader` загружает LoRA-модель и применяет её к базовой MODEL и CLIP
3. **Кодирование промптов** 📝 — два `CLIPTextEncode` преобразуют позитивный и негативный промпты в CONDITIONING (используют CLIP с применённой LoRA)
4. **Подготовка латентного пространства** ⬜ — `EmptyLatentImage` создает пустой латентный тензор (разрешение `1024x1024`)
5. **Сэмплирование** ⚙️ — `KSampler` выполняет генерацию с учётом LoRA-модифицированной модели
6. **Декодирование** 🔓 — `VAEDecode` преобразует латент в итоговое изображение
7. **Сохранение** 💾 — `SaveImage` сохраняет результат с префиксом `lora_output`

## Используемые ноды (переменные для настройки) 🎛️

### Загрузка моделей 📂

#### `CheckpointLoaderSimple` 📦
Загрузчик базовой модели.

| Поле | Описание | Пример значения |
|------|----------|-----------------|
| `ckpt_name` | Название файла чекпоинта | `RealVisXL_V5.0_Lightning_fp16.safetensors` |

#### `LoraLoader` 🎭
Загружает и применяет LoRA-модель.

| Поле | Описание | Пример значения |
|------|----------|-----------------|
| `lora_name` | Название LoRA-файла | `CPA.safetensors` |
| `strength_model` | Сила применения LoRA к модели __(влияет на стиль/персонажа)__ | `0.7` |
| `strength_clip` | Сила применения LoRA к CLIP __(влияет на понимание промптов)__ | `0.8` |

### Промпты 📝

#### `CLIPTextEncode` (позитивный) ✅
Что должно быть на изображении. Промпт должен соответствовать стилю LoRA.

| Поле | Описание | Пример значения |
|------|----------|-----------------|
| `text` | Позитивный промпт | `(masterpiece, best quality), 1girl, solo, cyberpunk anime style, neon lights, holograms, dark cityscape, detailed futuristic outfit, glowing eyes, vibrant colors, sharp lines, cinematic lighting` |

#### `CLIPTextEncode` (негативный) ❌
Чего не должно быть на изображении.

| Поле | Описание | Пример значения |
|------|----------|-----------------|
| `text` | Негативный промпт | `(worst quality, low quality:1.4), monochrome, greyscale, realistic, photo, photograph, **bad anatomy**, extra fingers, mutated hands, bad proportions` |

### Параметры генерации ⚙️

#### `EmptyLatentImage` 🖼️
Создает пустой латент.

| Поле | Описание | Пример значения |
|------|----------|-----------------|
| `width` | Ширина изображения | `1024` |
| `height` | Высота изображения | `1024` |
| `batch_size` | Количество изображений | `1` |

#### `KSampler` 🎲
Основной сэмплер.

| Поле | Описание | Пример значения |
|------|----------|-----------------|
| `seed` | Сид генерации | `851506285274005` или `randomize` |
| `steps` | Количество шагов | `8` |
| `cfg` | Масштаб классификатора | `2.5` |
| `sampler_name` | Алгоритм сэмплирования | `dpmpp_2m_sde` |
| `scheduler` | Расписание шума | `karras` |
| `denoise` | Сила денойзинга | `1` |

### Сохранение результатов 💾

#### `SaveImage` 💿
Сохраняет итоговое изображение.

| Поле | Описание | Пример значения |
|------|----------|-----------------|
| `filename_prefix` | Префикс имени файла | `lora_output` |

## Быстрый старт 🚀

1. Загрузите пайплайн в ComfyUI (файл `Lora (1).json`) 📂
2. В `CheckpointLoaderSimple` укажите путь к вашему чекпоинту __(без чекпоинта пайплайн не запустится)__ ⚠️
3. В `LoraLoader` укажите:
   - Название LoRA-файла (должен находиться в папке `ComfyUI/models/loras/`) 📁
   - `strength_model` — силу применения LoRA (обычно `0.5-1.0`) ⚖️
   - `strength_clip` — силу применения к CLIP (обычно `0.5-1.0`) ⚖️
4. Отредактируйте промпты в соответствии с вашей LoRA:
   - Позитивный промпт должен включать триггеры, специфичные для LoRA 🔑
   - Негативный промпт — стандартный для исключения артефактов ❌
5. Настройте разрешение в `EmptyLatentImage` (для SDXL рекомендуется `1024x1024` или `1216x832`) 📐
6. Нажмите "Queue Prompt" для генерации ▶️
7. Результат появится в папке `ComfyUI/output` с префиксом `lora_output` ✨

## Рекомендации по настройке LoRA ⚡

### Сила применения (`strength_model`)
| Значение | Эффект |
|----------|--------|
| `0.3-0.5` | Легкое влияние, едва заметные изменения |
| `0.6-0.8` | Среднее влияние, хорошо видимый эффект __(рекомендуется)__ |
| `0.9-1.2` | Сильное влияние, может нарушить композицию |

### Сила CLIP (`strength_clip`)
| Значение | Эффект |
|----------|--------|
| `0.5-0.7` | Умеренное влияние на понимание промптов |
| `0.8-1.0` | Сильное влияние, лучшее следование стилю LoRA __(рекомендуется)__ |

### Важные замечания ⚠️

- LoRA-модели должны быть совместимы с базовой моделью (SD1.5/SDXL) 🔄
- Для разных базовых моделей требуются соответствующие LoRA 🎯
- В промптах рекомендуется использовать ключевые слова, с которыми обучалась LoRA 🔑
- При слишком высоких значениях `strength_model` возможны артефакты и искажения 📉
- Некоторые LoRA требуют специфических триггеров в промпте (например, `<lora:name:weight>`) — проверяйте документацию к LoRA 📖
