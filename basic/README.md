# 🟢 Basic Workflows

Базовые пайплайны для генерации изображений — основа, с которой начинается любой проект в ComfyUI.

## 📋 Содержание раздела

| Пайплайн | Описание | Перейти |
|----------|----------|---------|
| **TXT2IMG** | Генерация изображения по текстовому описанию | [Открыть](./txt2img/README.md) |
| **Hires Fix** | Двухпроходная генерация для повышения качества и детализации | [Открыть](./hires-fix/README.md) |

---

## 🎯 Для чего нужны эти пайплайны

### ✍️ TXT2IMG
Самый простой и базовый пайплайн. Превращает текстовый промпт в изображение. Подходит для:
- Первого знакомства с ComfyUI
- Быстрого прототипирования идей
- Тестирования новых моделей и промптов

### 🔍 Hires Fix (High Resolution Fix)
Улучшает качество изображения за счёт двух проходов генерации:
1. **Первый проход** — создаётся композиция в низком разрешении (512×512)
2. **Второй проход** — изображение увеличивается и дорисовываются детали (1024×1024)

Идеально подходит для:
- Получения高质量 (высококачественных) изображений
- Проработки мелких деталей (текстуры, волосы, глаза)
- Финальной обработки результатов из других пайплайнов

---

## ⚙️ Общие настройки для всех пайплайнов

Все пайплайны в этом разделе оптимизированы для работы с **RealVisXL V5.0 Lightning** (SDXL).

| Параметр | Значение | Пояснение |
|----------|----------|-----------|
| **Базовая модель** | RealVisXL V5.0 Lightning | Фотореалистичная SDXL-модель |
| **Разрешение** | 1024×1024 | Родной размер для SDXL |
| **Sampler** | `dpmpp_sde_karras` | Оптимален для Lightning-моделей |
| **CFG Scale** | 2.0–2.5 | Низкое значение для естественности |
| **Steps** | 8–10 | Lightning-модели требуют мало шагов |

## 📦 Требования

### Плагины
Для работы с этими пайплайнами дополнительные плагины **не требуются** — всё работает на базовых нодах ComfyUI.

### Модели

| Модель | Назначение | Ссылка | Куда положить |
|--------|------------|--------|---------------|
| **RealVisXL V5.0 Lightning** | Базовая модель для генерации | [Hugging Face](https://huggingface.co/SG161222/RealVisXL_V5.0_Lightning) | `models/checkpoints/` |

> 💡 **Совет:** Если вы используете другую SDXL-модель, просто замените её название в ноде `CheckpointLoaderSimple` — пайплайны универсальны.

## 🚀 Как начать работу

### Шаг 1. Установите ComfyUI
Если ещё не установили — скачайте с [официального GitHub](https://github.com/comfyanonymous/ComfyUI).

### Шаг 2. Скачайте модель RealVisXL V5.0 Lightning
Положите файл `.safetensors` в папку `ComfyUI/models/checkpoints/`.

### Шаг 3. Выберите нужный пайплайн
- Для простой генерации — [**TXT2IMG**](./txt2img/)
- Для улучшения качества — [**Hires Fix**](./hires-fix/)

### Шаг 4. Загрузите JSON в ComfyUI
Просто перетащите файл `workflow.json` в окно браузера с открытым ComfyUI.

### Шаг 5. Настройте промпт и запустите
Измените текст в нодах `CLIPTextEncode` (позитивный и негативный промпты) и нажмите **Queue Prompt**.

---

## 📌 Важные замечания

- ⚡ **Lightning-модели** требуют мало шагов (5–10). Не ставьте 30–40 — это избыточно.
- 🎨 Для TXT2IMG используйте **denoise = 1.0**
- 🔄 Для Hires Fix во втором проходе ставьте **denoise = 0.4–0.5**
- 💾 При первом запуске модель загружается в VRAM — может быть пауза 10–20 секунд

## 🐛 Возможные проблемы и их решение

### ❌ Изображение размытое
**Причина:** Слишком мало шагов или низкий CFG.  
**Решение:** Увеличьте `steps` до 10–12 или поднимите `cfg` до 3.0.

### ❌ Артефакты, искажения
**Причина:** Слишком высокий CFG или неподходящий sampler.  
**Решение:** Уменьшите `cfg` до 2.0, смените sampler на `dpmpp_sde` или `dpmpp_2m`.

### ❌ Долгая генерация
**Причина:** Слишком много шагов или не оптимизировано под Lightning.  
**Решение:** Уменьшите `steps` до 8–10.
