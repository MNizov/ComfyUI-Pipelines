# Hires Fix (2-проходная генерация) 🔄✨

Пайплайн для двухпроходной генерации изображений с повышением качества и разрешения (Hires Fix). Первый проход создает базовое изображение, второй — улучшает детализацию и увеличивает разрешение. 🖼️➡️🖼️✨

## Описание работы пайплайна 🔄

Пайплайн реализует технику Hires Fix для получения высокодетализированных изображений:

### Первый проход (базовая генерация) 🎨
1. **Загрузка модели** 📥 — `CheckpointLoaderSimple` загружает базовую модель
2. **Кодирование промптов** 📝 — два `CLIPTextEncode` создают позитивное и негативное CONDITIONING для первого прохода
3. **Подготовка латентного пространства** ⬜ — `EmptyLatentImage` создает пустой латент низкого разрешения (512x512)
4. **Сэмплирование (проход 1)** ⚙️ — `KSampler` генерирует латентное представление изображения
5. **Декодирование (проход 1)** 🔓 — `VAEDecode` преобразует латент в изображение
6. **Сохранение промежуточного результата** 💾 — `SaveImage` сохраняет изображение первого прохода (префикс `pass1`)

### Подготовка ко второму проходу 🔧
7. **Кодирование для апскейла** 🔄 — `VAEEncode` преобразует полученное изображение обратно в латентное пространство
8. **Апскейл латента** 📏 — `LatentUpscale` увеличивает разрешение латентного тензора (до 720x720)

### Второй проход (апскейл и детализация) ✨
9. **Кодирование промптов (проход 2)** 📝 — два `CLIPTextEncode` создают CONDITIONING для второго прохода (можно использовать те же или уточненные промпты)
10. **Сэмплирование (проход 2)** ⚙️ — второй `KSampler` выполняет денойзинг с пониженной силой (`denoise` ~0.4-0.5) для добавления деталей
11. **Декодирование (проход 2)** 🔓 — `VAEDecode` преобразует улучшенный латент в итоговое изображение высокого разрешения
12. **Сохранение финального результата** 💾 — `SaveImage` сохраняет итоговое изображение (префикс `pass2_hires`)

## Используемые ноды (переменные для настройки) 🎛️

### `CheckpointLoaderSimple` 📦
Загрузчик базовой модели. Используется для всех проходов.

| Поле | Описание | Пример значения |
|------|----------|-----------------|
| `ckpt_name` | Название файла чекпоинта | `RealVisXL_V5.0_Lightning_fp16.safetensors` |

### `CLIPTextEncode` (проход 1, позитивный) ✅
Позитивный промпт для первого прохода.

| Поле | Описание | Пример значения |
|------|----------|-----------------|
| `text` | Позитивный промпт | `portrait of a beautiful young woman, soft studio lighting, detailed face, natural skin texture, sharp focus, 8k, photorealistic` |

### `CLIPTextEncode` (проход 1, негативный) ❌
Негативный промпт для первого прохода.

| Поле | Описание | Пример значения |
|------|----------|-----------------|
| `text` | Негативный промпт | `blurry, low quality, cartoon, painting, illustration, anime, sketch, watermark, text, ugly, deformed, bad anatomy, extra fingers, fused fingers, mutation, bad proportions, cropped, worst quality, jpeg artifacts, out of frame, duplicate, morbid, mutated, distorted, bad hands, missing fingers, extra limbs, malformed limbs` |

### `EmptyLatentImage` 🖼️
Создает пустой латент для первого прохода.

| Поле | Описание | Пример значения |
|------|----------|-----------------|
| `width` | Ширина (первый проход) | `512` |
| `height` | Высота (первый проход) | `512` |
| `batch_size` | Количество изображений | `1` |

### `KSampler` (проход 1) 🎲
Первый сэмплер для базовой генерации.

| Поле | Описание | Пример значения |
|------|----------|-----------------|
| `seed` | Сид генерации | `658744807723594` или `randomize` |
| `steps` | Количество шагов | `30` |
| `cfg` | Масштаб классификатора | `6.2` |
| `sampler_name` | Алгоритм сэмплирования | `dpmpp_2m` |
| `scheduler` | Расписание шума | `karras` |
| `denoise` | Сила денойзинга | `1` |

### `VAEEncode` 🔄
Кодирует изображение первого прохода обратно в латент. Не требует настройки.

### `LatentUpscale` 📏
Увеличивает разрешение латентного тензора.

| Поле | Описание | Пример значения |
|------|----------|-----------------|
| `upscale_method` | Метод апскейла | `nearest-exact` |
| `width` | Целевая ширина | `720` |
| `height` | Целевая высота | `720` |
| `crop` | Режим обрезки | `center` |

### `CLIPTextEncode` (проход 2, позитивный) ✅✨
Позитивный промпт для второго прохода (можно уточнить для добавления деталей).

| Поле | Описание | Пример значения |
|------|----------|-----------------|
| `text` | Позитивный промпт | `portrait, highly detailed face, sharp focus, natural skin texture, soft lighting, 8k` |

### `CLIPTextEncode` (проход 2, негативный) ❌✨
Негативный промпт для второго прохода.

| Поле | Описание | Пример значения |
|------|----------|-----------------|
| `text` | Негативный промпт | `blurry, low quality, cartoon, painting, illustration, anime, sketch, watermark, text, ugly, deformed, bad anatomy, extra fingers, fused fingers, mutation, bad proportions, cropped, worst quality, jpeg artifacts, out of frame, duplicate, morbid, mutated, distorted, bad hands, missing fingers, extra limbs, malformed limbs, blurry background, out of focus background, low quality background, simple background, plain background, blurry clothes, blurry background, simple background, low detail clothes, fuzzy` |

### `KSampler` (проход 2) 🎲✨
Второй сэмплер для улучшения детализации.

| Поле | Описание | Пример значения |
|------|----------|-----------------|
| `seed` | Сид генерации | `555127526509379` или `randomize` |
| `steps` | Количество шагов | `26` |
| `cfg` | Масштаб классификатора | `5` |
| `sampler_name` | Алгоритм сэмплирования | `dpmpp_sde` |
| `scheduler` | Расписание шума | `karras` |
| `denoise` | Сила денойзинга __(важно! 0.3-0.5 для сохранения композиции)__ | `0.43` |

### `VAEDecode` (проход 1 и 2) 🔓
Декодирует латенты в изображения. Не требует настройки.

### `SaveImage` (проход 1) 💾
Сохраняет промежуточное изображение.

| Поле | Описание | Пример значения |
|------|----------|-----------------|
| `filename_prefix` | Префикс имени файла | `pass1` |

### `SaveImage` (проход 2) 💾✨
Сохраняет финальное изображение.

| Поле | Описание | Пример значения |
|------|----------|-----------------|
| `filename_prefix` | Префикс имени файла | `pass2_hires` |

## Быстрый старт 🚀

1. Загрузите пайплайн в ComfyUI (файл `Hires Fix (2-проходная генерация).json`) 📂
2. В ноде `CheckpointLoaderSimple` укажите путь к вашему чекпоинту __(без чекпоинта пайплайн не запустится)__ ⚠️
3. Настройте позитивные и негативные промпты для обоих проходов ✏️
4. При необходимости измените разрешение:
   - Первый проход: `512x512` в `EmptyLatentImage` 📐
   - Второй проход: целевое разрешение в `LatentUpscale` (например, `1024x1024` или `1280x720`) 📏
5. Настройте `denoise` во втором `KSampler`:
   - `0.3-0.4` — минимальные изменения, сохранение композиции
   - `0.5-0.6` — более сильные изменения, больше деталей
6. Нажмите "Queue Prompt" для генерации ▶️
7. Результаты появятся в папке `ComfyUI/output`:
   - `pass1_XXXXX.png` — изображение после первого прохода 🖼️
   - `pass2_hires_XXXXX.png` — финальное изображение после апскейла ✨
